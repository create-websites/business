INNER JOIN

Postgres looks at the user_id in the Order row, matches it to the id in the Users table, and creates a combined result.

SQL Query:

SELECT Users.name, Orders.total
FROM Orders
JOIN Users ON Orders.user_id = Users.id
WHERE Orders.id = ...;

Beyond Basic SQL

Extend functionality without leaving DB

One of Postgres's superpowers is its extension system.

  • PostGIS:

    Adds support for geographic objects. Run queries like "Find all coffee shops within 500 meters of this GPS point."

  • pgvector:

    Turn Postgres into a Vector Database. Store embeddings and perform cosine similarity search right next to your relational data.

More than just B-Trees

Postgres supports many index types for different data patterns.

B-Tree: Standard index for sorting and equality.
GIN (Generalized Inverted Index): Perfect for indexing JSONB keys or Full-Text Search vectors.
GiST: Used for geometric data (PostGIS) and nearest-neighbor search.

Modern SQL Features

-- Common Table Expressions (CTEs) WITH RegionalSales AS ( SELECT region, SUM(amount) as total_sales FROM orders GROUP BY region ) SELECT * FROM RegionalSales WHERE total_sales > (SELECT AVG(total_sales) FROM RegionalSales);

Features like CTEs, Window Functions, and Recursive Queries make complex analytics possible directly in SQL.
